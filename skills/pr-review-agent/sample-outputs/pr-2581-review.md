### 📋 Summary of Changes

This PR adds a new Claude Code skill (`changelog-generator`) that generates a structured CHANGELOG.md from git history. It includes a Bash script that auto-categorizes commits using Conventional Commits prefixes (feat → Added, fix → Fixed, etc.) and handles repos both with and without tags. The submission also includes a complementary `destructive-command-blocker` hook as a separate deliverable in the same fork.

### ⚠️ Identified Risks

- The script uses `grep -qE` with lowercase-converted commit messages — this may misclassify non-English or mixed-case commit formats
- No escaping of special characters in commit messages — a commit message containing `"` or backticks could break the CHANGELOG.md output format
- The script only handles the first conventional commit prefix, but some messages have multiple prefixes or scopes (e.g., `feat(api): ...`)
- Both the CHANGELOG skill and the destructive command blocker are in a single PR, making it harder to review independently

### 💡 Improvement Suggestions

- Add shellcheck validation and use `printf` instead of raw string interpolation for safer output
- Support scoped conventional commits: `feat(scope):` → Added
- Consider adding a `--dry-run` flag to preview output before writing
- Split the two skills into separate PRs for cleaner review
