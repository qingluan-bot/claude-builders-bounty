### 📋 Summary of Changes

This PR introduces an opinionated CLAUDE.md template for Next.js 15 + SQLite (better-sqlite3) SaaS projects. The template covers technology choices, directory structure conventions, naming rules, SQL migration standards, component patterns, anti-patterns, and git workflow. Each convention includes a rationale explaining why a particular approach was chosen over alternatives.

### ⚠️ Identified Risks

- Strong opinions (e.g., "no Prisma", "no GraphQL", "no Redux") may not suit all team preferences or project requirements — the template could be rejected by teams with different tech stacks
- The CLAUDE.md may become stale quickly as Next.js and ecosystem dependencies evolve rapidly
- SQLite-specific conventions don't apply to projects using PostgreSQL or other databases, limiting reuse
- No versioning or changelog for the template itself — teams won't know when conventions are updated

### 💡 Improvement Suggestions

- Add a "When to Deviate" section explaining under what conditions each rule could be broken
- Include migration scripts or codemods for transitioning existing projects to the conventions
- Consider a companion validation script that checks a project against the CLAUDE.md rules
- Use environment variable examples rather than hardcoded paths for database location
